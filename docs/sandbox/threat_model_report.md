# Code Sandbox 威胁建模报告

## 1. 资产清单
- 执行节点主机资源（CPU/内存/磁盘/网络）。
- 平台密钥与凭据（签名密钥、CI 凭据）。
- 审计日志与任务元数据。
- 业务系统 API 与内部网络。

## 2. 攻击面
- 用户提交代码（核心不可信输入）。
- 容器运行时与镜像供应链。
- CI/CD 流水线与依赖包。
- 审计日志与对象存储读写接口。

## 3. STRIDE 分析
- Spoofing: 伪造用户身份提交任务。
  - 控制: 强制 user_id + 鉴权网关签名。
- Tampering: 篡改执行代码或日志。
  - 控制: 代码哈希签名、不可变日志存储。
- Repudiation: 提交者否认执行行为。
  - 控制: 审计日志落地（用户ID+哈希+时间）。
- Information Disclosure: 越权读取宿主机或敏感文件。
  - 控制: 只读根文件系统、路径校验、系统调用限制。
- Denial of Service: 死循环、fork 炸弹、资源耗尽。
  - 控制: CPU/内存/pids/超时限制 + 熔断。
- Elevation of Privilege: 容器逃逸、提权。
  - 控制: cap-drop、no-new-privileges、seccomp、AppArmor。

## 4. 风险分级
- 高风险:
  - 容器逃逸
  - 任意命令执行
  - 横向网络扫描
- 中风险:
  - 非授权文件访问
  - 日志注入
- 低风险:
  - 非敏感资源误用

## 5. 缓解与残余风险
- 已缓解:
  - 绝大多数脚本级命令注入与网络外联。
- 残余:
  - 0day 内核漏洞导致容器隔离失效。
  - 供应链镜像投毒。
- 额外建议:
  - 镜像签名验证（cosign）
  - 节点最小化 OS + 自动补丁
  - eBPF 行为基线告警
