# Code Sandbox 性能压测报告

## 1. 测试目标
- 冷启动延迟: <= 800ms
- 并发能力: 1000 实例
- 扩容阈值: CPU 利用率 <= 80% 触发横向扩容

## 2. 测试环境
- 节点规格: 8 vCPU / 32 GB RAM
- 容器镜像: `python:3.11-alpine`
- 执行任务: `print("hello")` 与 CPU/IO 混合任务
- 压测工具: k6 + 自定义执行器 API

## 3. 指标定义
- p50/p95/p99 启动耗时
- 容器创建失败率
- 平均执行时长
- 节点 CPU/内存利用率
- 熔断触发次数

## 4. 基线结果（样例）
- 冷启动:
  - p50: 420ms
  - p95: 760ms
  - p99: 905ms（超标，建议镜像预热）
- 并发:
  - 1000 并发下成功率 99.2%
  - 平均执行时长 1.4s
- 资源:
  - CPU 峰值 78%
  - 内存峰值 71%

## 5. 优化建议
- 预热容器池减少冷启动抖动。
- 拆分队列为短任务和长任务通道。
- 将审计日志异步批量写入对象存储。
