# Code Sandbox 安全渗透测试报告

## 1. 测试范围
- 代码提交接口
- 静态扫描策略
- 容器隔离参数（seccomp/capabilities/AppArmor）
- 风险行为检测链路
- 审计日志完整性

## 2. 测试方法
- 黑盒:
  - 构造恶意脚本（反弹 shell、fork bomb、外联）
- 灰盒:
  - 校验容器启动参数、策略配置、日志字段

## 3. 攻击用例与结论
- 文件越权访问: `open('/etc/passwd')`
  - 结果: 被静态扫描与运行时规则阻断。
- 网络外联: `socket` + HTTP 请求
  - 结果: `network none` + 风险检测阻断。
- 子进程逃逸: `subprocess.Popen`
  - 结果: 静态扫描直接拒绝。
- 敏感 API: `eval/exec`
  - 结果: 静态扫描拒绝。

## 4. 发现问题
- 低风险:
  - 默认签名密钥可通过环境变量覆盖，但需在生产强制非默认值。

- 上线前校验 `SANDBOX_SIGNING_KEY` 复杂度。
- 增加镜像签名验签与漏洞扫描门禁。
